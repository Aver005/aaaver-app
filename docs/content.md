# Где менять контент

Базы данных для контента нет — всё лежит в коде, меняется правкой файла
и пересборкой (`docker compose up --build -d`).

## Тексты интерфейса (RU/EN)

`src/shared/i18n/dict.ts` — единый словарь: hero, заголовки секций,
форма, футер. Два объекта, `ru` и `en`, структура у них одинаковая,
TypeScript не даст забыть перевод.

## Проекты

`src/entities/project/model/data.ts`. У проекта есть:

```ts
{
    id: 'pocacall',
    title: 'Pocacall',
    url: 'https://…',
    image: '/projects/pocacall.png',   // скриншот в public/projects/
    description: { ru: '…', en: '…' },
    tags: ['WebRTC', 'React'],
}
```

Скриншоты лежат в `public/projects/`, пропорция карточки — 16:10,
оптимальная ширина ~1200px. Порядок в массиве = порядок на странице,
карточки автоматически чередуют лево/право.

## Опыт работы

`src/entities/experience/model/data.ts`. Поле `to: null` означает
«по настоящее время». Порядок в массиве = порядок на странице
(сверху — свежее).

## Стек

`src/entities/skill/model/data.ts` — группы и их элементы. Эти же
элементы крутятся в бегущей строке под hero.

## Open Source

Секция сама тянет репозитории с GitHub (см. `server/routes/github.ts`):
не-форки, отсортированы по последнему пушу. Менять руками ничего не надо.

Если нужно скрыть репозиторий — проще всего расширить фильтр в
`fetchFromGithub()`. Снимок для фолбэка (показывается, пока грузится
живой список) лежит в `src/entities/repo/model/fallback.ts` — его стоит
изредка освежать.

## Контакты и ссылки

`src/shared/config/site.ts` — телеграм, почта, GitHub. Используются
в секции «Контакт», футере и шапке.

## SEO-мелочи

`index.html` — title, description, OG-теги, JSON-LD. Фавиконки в
`public/`.
